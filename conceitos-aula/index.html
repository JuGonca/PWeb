Links

Delimita os tipor de ligações:

  • De uma página para algum outro site.
  • De uma página para outra página em um mesmo site.
  • De uma parte para outra em uma mesma página.

Atributo HREF

Adiciona o endereço para o qual o arquivo deve ser redirecionado, arquivo de vídeo, arquivo de imagem.

  • Caminho absoluto - Endereço completo.
  • Caminho relativo - Endereço referente aonde o arquivo estava.

Atributo TITLE

Funciona da mesma forma que as outras tags. É o título da tag, uma descrição do que aquele elemento faz.

O item é apresentado em forma de balão.

Target

O atributo target na tag de link, indica para onde será redirecionado a página do link quando você clicar nele.

  • _blanc - O link será aberto em uma outra aba no brownser.
  • _self - O link será aberto na mesma janela.
  • _top - O link é aberto na mesma janela sem frames.
  • _parent - O link é aberto no frameset principal.

Imagens

É a tag que adiciona uma imagem no site.

Existe tanto a tag img como a tag figure. Ambas tem o mesmo prosósito, a diferença é que a tag img apenas exine a imagem, e a tag figure junta a imagem com a legenda, fazendo aquele elemento visual se tornar um só.

Dentro da tag img temos os seguintes atributos:

  • src - É o endereço da imagem. Pode ser um link direto ou o caminho para alguma pasta do projeto
  • alt - Mostra uma "legenda" quando colocamos o cursor em cima da imagem.
  • title - Tem o mesmo sentido que na tag de âncora, adicionar um título para a imagem de forma sintática.
  • width - Largura da imagem.
  • height - Altura da imagem.

Base64

A base64 é um método de codificação de dados usado em transferências na internet.

O método é constituído por 64 caracteres, sendo eles [A-Za-z0-9, / e + que deram origem ao seu nome.

Exemplo

  • Texto original: Hello Word
  • Texto convertido: aGVsbG8gd29ybGQK

Esse método, como todos os outros, possui vantagens e desvantagens.

  • A imagens podem começar a carregar mesmo que o site em si não tenha terminado o download.
  • As imagens em base64 pode deixar o site mais lento por acabar aumentando muito o seu tamanho.
  • A vantagem está em não ter que abrir um outra conexão só para a imagem.

Tabelas

Table - Usar a tag apenas quando fizer sentido os dados serem organizados em formato de linhas ou colunas, como tabela de dados, calendários, tabela de preços, entre outros.

Em outros casos, é melhor mudar no css a organização utilizando o display: table.

Dados tabulares

  • caption - Insere o cabeçalho de título na tabela
  • th - titulos de linhas ou colulas
  • thead, tbody, tfoot - Serve para agrupar linhas de uma tabelas e definílas. O sentido de cada uma é parecido como no corpo da página web.
  • tr - Define as linahs das tabelas
  • td - Cria as colunas desejadas e insere o conteúdo de cada linha.

Elementos estruturais

O conjunto de tags em html é muito grande, porém limitiado, e quando não temas uma tag para usar em um momento específico, temos duas opções de tags: div, e span. Ambas são tags sem significado que servem para agrupar algum conteúdo da página.

  • Div - Div vem de divisão, ela serve para agrupar links, imagens, o conteúdo que desejar. Por padrão, não possui nenhum estilo, sendo possível alterar pelo css. A div deve ser utilizada principalmente quando não se precitar ter um significado semântico, apenas estilização dos elementos.
  • Span - O span, diferente da div que trabalha com um "bloco de código", o span trabalha apenas com alguma parte da tag. Utilizamos ids e classes para diferenciar os elementos html no css, portém com o span, podemos mudar apenas uma parte daquela tag.

Html semântico

O html semântico trás significado para os elementos/tags sem impactar no visual. Portanto são estremamente importantes para leitores não visuais, pois trazem sentigo para aquela página.

  • main - define o conteúdo principal do body. O main está relacionado com o tema principal da página, e deve ser usado apenas uma vez.
  • nav - É o bloco principal de navegações da página. Fica normalmente o menu.
  • aside - É um bloco de informações secundárias da página, servindo como um complemento para aquele conteúdo. Ex: Citações, referências, ou pode ser até mesmo um sumário para os tópicos da página.
  • header - Refere-se ao cabeçãlho de uma seção específica. Podendo ficar somente no main, ou em uma section por exemplo
  • foot - Refere-se ao rodapé da página.
  • article - é o conteúdo em si, como noticia ou post.

Formulários

Os formulários são utilizados para insersão de dador e interação do usuário com o navegador. Tudo o que estiver dentro da tag form será compreendido como fazendo parte do formulário.

  • label - É o título ou rótulo do campo.
  • input - É a tag do campo de entrada, nela serão inseridos os dados para a interação com o formulário. Para saber o tipo de dado, utilizamos o atributo type dentro da tag. Exemplo:

Atributos dos formulários

  • action - indica para onde os dados informados no input serão submetidos.
  • method - Especifica o método de submissão usado
    • get - default, o conteúdo do formulário é anexado no final da url informada no action.
    • post - o conteúdo do formulário é identificado, e em blocos é enviado até a url informada.
  • enctype - define como os dados do formulário são codificados antes de serem enviados ao servidor. É normalmente usado quando o method é post.
  • fieldset - Serve para agrupar um conjunto de campos de um formulário que estão interligados.
  • legend - é a legenda do campo.
